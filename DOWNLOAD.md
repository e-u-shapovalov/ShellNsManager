# Как скачать ShellNsManager

Этот файл для обычного пользователя, который хочет просто скачать и запустить программу.

## Коротко

Скачивайте готовую программу из **GitHub Releases**, а не исходный код.

Правильный файл обычно называется примерно так:

- `ShellNsManager-v1.0-win64.zip`
- `ShellNsManager.exe`

Неправильные файлы для обычного пользователя:

- `Source code (zip)`
- `Source code (tar.gz)`
- зеленая кнопка `Code -> Download ZIP`

Эти варианты нужны разработчикам. Они не являются готовой программой для запуска.

## Где находится GitHub Releases

1. Откройте страницу репозитория ShellNsManager на GitHub.
2. Найдите ссылку **Releases**. Обычно она находится справа на странице проекта или рядом с информацией о репозитории.
3. Откройте самый новый релиз.
4. Найдите блок **Assets**.
5. Скачайте архив или `.exe` с названием `ShellNsManager`.

Прямая ссылка из README: [последний релиз](../../releases/latest).

## Как запустить

1. Если скачали `.zip`, нажмите правой кнопкой мыши и выберите **Extract All... / Извлечь все...**.
2. Откройте распакованную папку.
3. Запустите `ShellNsManager.exe`.
4. Подтвердите запрос контроля учетных записей Windows.

Программа требует права администратора, потому что меняет настройки Проводника в реестре Windows.

## Если Windows показывает предупреждение

Windows SmartScreen может показать предупреждение, потому что `.exe` не подписан коммерческим сертификатом.

Что делать:

- убедитесь, что скачали файл именно из **Releases** этого репозитория;
- не запускайте `.exe`, полученный из случайных сайтов и архивов;
- если вы разработчик, можете собрать программу самостоятельно из исходников;
- если доверяете источнику, в окне SmartScreen можно выбрать **More info / Подробнее** и затем **Run anyway / Выполнить в любом случае**.

## Если Releases пустой

Если на странице Releases нет готового `.zip` или `.exe`, значит публичный релиз еще не создан.

В этом случае:

- обычному пользователю лучше дождаться релиза;
- разработчик может собрать программу локально через `build.bat`;
- автору проекта нужно создать GitHub Release и загрузить туда готовый архив.

Инструкция для разработчика и релиза: [INSTALL.md](INSTALL.md) и [RELEASES.md](RELEASES.md).
