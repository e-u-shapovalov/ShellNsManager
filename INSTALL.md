# Установка, запуск и сборка ShellNsManager

## Для обычного пользователя

ShellNsManager - portable-программа. Установщик не нужен.

1. Скачайте готовый архив или `.exe` из [GitHub Releases](../../releases/latest).
2. Если это архив, распакуйте его.
3. Переместите папку в удобное место, например `C:\Tools\ShellNsManager`.
4. Запустите `ShellNsManager.exe`.
5. Подтвердите UAC-запрос администратора.

Не запускайте программу прямо из архива. Сначала распакуйте ее, чтобы рядом с `.exe` могли создаваться папки `backups` и `lang`.

## Системные требования

- Windows 10 или Windows 11.
- Права администратора.
- Обычный Windows Explorer (`explorer.exe`).
- Дополнительные runtime-пакеты не требуются.

## Что создается рядом с программой

- `backups\YYYYMMDD-HHMMSS\` - `.reg`-бэкапы перед изменениями.
- `lang\en.lang` - английский языковой файл, создается при первом переключении языка на English.

Папку `backups` лучше не удалять сразу после настройки: она нужна для отката.

## Как откатить изменение

1. Откройте папку `backups`.
2. Найдите подпапку с датой и временем нужной операции.
3. Импортируйте подходящий `.reg`-файл двойным кликом или через `regedit`.
4. Перезапустите Проводник из ShellNsManager или перезагрузите Windows.

Если восстановление затрагивает защищенные системные ключи, запускайте `regedit` от имени администратора.

## Для разработчиков

Проект не использует npm, CMake, NuGet или сторонние библиотеки. Сборка выполняется стандартными инструментами Microsoft C++.

### Требования

- Windows.
- Visual Studio 2022 Build Tools.
- Компонент **Desktop development with C++**.
- Windows SDK.

`build.bat` ожидает `vcvars64.bat` по пути:

```text
C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools\VC\Auxiliary\Build\vcvars64.bat
```

Если Visual Studio установлена в другое место, поправьте переменную `VCVARS` в `build.bat`.

### Сборка

```bat
git clone <URL этого репозитория>
cd ShellNsManager
build.bat
```

После успешной сборки:

- `ShellNsManager.exe` появится в корне проекта;
- `.obj` и `resource.res` будут лежать в `build\`.

### Структура проекта

```text
src\main.cpp        - главное окно, меню, дерево, команды UI
src\registry.cpp    - чтение/запись реестра, бэкапы, изменение namespace-узлов
src\nsmodel.cpp     - разрешение имен, иконок и описаний Shell Namespace элементов
src\add_dialog.cpp  - диалог добавления пользовательской папки
src\hide_dialog.cpp - диалог скрытия встроенных элементов
src\loc.cpp         - русская/английская локализация
src\resource.rc     - ресурсы Win32, диалоги, manifest, иконка
build.bat           - сборка через Visual Studio Build Tools
```

### Проверка перед релизом

1. Соберите `ShellNsManager.exe` через `build.bat`.
2. Запустите на тестовой Windows 10/11.
3. Проверьте добавление папки в `HKCU`.
4. Проверьте удаление созданного узла и восстановление из `backups`.
5. Проверьте скрытие/возврат «Главная» и «Быстрый доступ».
6. Убедитесь, что `.reg`-бэкапы создаются.
7. Сделайте архив релиза и загрузите его в GitHub Releases.
